# 规则3: 质量检查

## 目标
确保添加的链接符合质量标准，无重复、格式正确。

## 执行步骤

### 3.1 检查重复链接
运行以下命令检查完全重复的URL：
```bash
cd /Users/zhujin05/data/seo/code/seo-nav
grep -o 'href="[^"]*"' index.html | \
  sed 's/href="//;s/"$//' | \
  grep -E '^https?://' | \
  sort | uniq -c | \
  awk '$1 > 1 {print "重复" $1 "次:", $2}'
```

**规则**：
- ❌ **禁止**：完全相同的URL重复出现
- ✅ **允许**：同一网站的不同页面（不同路径，如 `/page1` 和 `/page2`）

### 3.2 处理重复链接
如果发现重复：
1. 保留更具体的名称（如"站长工具关键词"而非"站长工具"）
2. 删除其他重复项
3. 确保删除后表格结构完整

### 3.3 格式检查
检查所有链接：
- [ ] URL以 `https://` 或 `http://` 开头
- [ ] 包含 `target="_blank"` 属性
- [ ] 显示名称不为空
- [ ] HTML格式正确

### 3.4 分类检查
- [ ] 链接分类合理
- [ ] 相关工具在同一分类
- [ ] 分类顺序符合逻辑

### 3.5 表格结构检查
- [ ] 每行4个单元格
- [ ] rowspan值正确
- [ ] 无未闭合标签
- [ ] 表格结构完整

### 3.6 使用linter检查
```bash
read_lints index.html
```

## 验证清单
- [ ] 无重复链接
- [ ] 所有链接格式正确
- [ ] 表格结构完整
- [ ] 无linter错误
- [ ] 分类合理

## 注意事项
- 检查时注意区分"完全重复"和"同一网站不同页面"
- 删除重复后要验证表格结构
- 确保所有更改都已保存
