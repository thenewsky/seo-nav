# SEO导航站 - 新增链接操作规范

## 命令名称
`add-seo-links` - 向SEO导航站添加新链接的标准化流程

## 操作流程

### 1. 准备工作
- 确保当前在 `main` 分支且工作区干净
- 确认要添加的链接列表和分类

### 2. 创建分支
- 从 `main` 创建稳定版本分支（可选，仅在需要时）
- 从 `main` 创建功能开发分支：`feature/add-links-YYYYMMDD`

### 3. 添加链接规则

#### 3.1 链接分类
现有分类模块：
- `SEO社区/学习资源` - 社区和论坛
- `SEO工具/关键词研究` - 关键词研究工具
- `SEO工具/网站分析` - 网站流量和分析工具
- `SEO工具/竞争分析` - 竞争分析和关键词难度
- `SEO教程/最佳实践` - SEO学习资源
- `域名工具` - 域名相关工具
- `AI工具` - AI相关工具
- `开发工具` - 开发辅助工具

#### 3.2 链接格式
```html
<td><a href="https://example.com" target="_blank">显示名称</a></td>
```

#### 3.3 添加规则
1. **分类选择**：根据链接功能选择合适分类，如无合适分类可创建新分类
2. **位置选择**：优先填充空单元格（`<td></td>`），如无空位则新增行
3. **命名规范**：
   - 使用简洁明确的中文名称
   - 同一网站的不同页面应区分（如：`SEO Box` 和 `SEO Box关键词`）
   - 工具入口可标注功能（如：`UE (Semrush/Similarweb)`）

#### 3.4 表格结构维护
- 每行4个单元格（`<td>`）
- 分类标题使用 `<th rowspan="N">`，N为行数
- 空单元格使用 `<td></td>`
- 保持表格结构完整

### 4. 质量检查

#### 4.1 重复检查
- **禁止**：完全相同的URL重复出现
- **允许**：同一网站的不同页面（不同路径）
- 检查命令：`grep -o 'href="[^"]*"' index.html | sed 's/href="//;s/"$//' | grep -E '^https?://' | sort | uniq -c | awk '$1 > 1'`

#### 4.2 格式检查
- 所有链接必须包含 `target="_blank"`
- URL必须以 `https://` 或 `http://` 开头
- 显示名称不能为空

### 5. 提交和合并

#### 5.1 提交更改
```bash
git add index.html
git commit -m "feat: 添加新链接到SEO导航站

- 添加 [链接名称1] ([分类])
- 添加 [链接名称2] ([分类])
- 更新 [其他改动]"
```

#### 5.2 合并到main
```bash
git checkout main
git merge feature/add-links-YYYYMMDD --no-ff
```

#### 5.3 创建Release分支
```bash
git checkout -b release/vX.Y
```

#### 5.4 打Tag
```bash
git tag -a vX.Y.Z -m "vX.Y.Z: 添加新链接到SEO导航站"
```

## 版本号规则

- **主版本号 (X)**：重大结构调整或大量链接更新
- **次版本号 (Y)**：新增分类或中等规模更新
- **修订号 (Z)**：少量链接添加或修复

## 注意事项

1. **不要删除已有链接**，除非明确要求
2. **保持分类逻辑清晰**，相关工具放在同一分类
3. **优先使用空单元格**，避免频繁调整rowspan
4. **提交信息要清晰**，说明添加了哪些链接
5. **合并前检查**：确保无重复链接和格式错误

## 示例

### 添加单个链接
```bash
add-seo-links --url "https://example.com" --name "示例工具" --category "SEO工具/关键词研究"
```

### 批量添加链接
```bash
add-seo-links --file links.txt
```

### 仅检查重复
```bash
add-seo-links --check-duplicates
```
